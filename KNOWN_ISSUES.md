# 알려진 이슈 (Known Issues)

## 1. 목(目) 하위 구조 미파싱

국가법령정보센터 API의 XML 구조는 `조문단위 > 항 > 호 > 목`까지만 정형화되어 있다.
목 아래에 존재하는 하위 항목(반괄호 `1)`, `2)` 등)은 별도 XML 요소가 아닌
`목내용` 텍스트에 포함되어 있어, 현재 파서는 이를 구조적으로 분리하지 않는다.

**영향**: 도로교통법 제2조 제17호 가목 등에서 반괄호 항목이 본문 텍스트로 출력됨.
공백 정규화로 가독성은 개선되었으나, 들여쓰기 등 구조적 표현은 불가.

**해결 방향**: `목내용` 텍스트에서 `N)` 패턴을 정규식으로 분리하여 하위 리스트로
변환하는 후처리 추가. 단, API 원본의 일관성 검증이 선행되어야 함.

## 2. 호(號) 번호 체계와 API 반환값 불일치

`호번호` 필드의 형식이 API 응답마다 다를 수 있다 (예: `"1"`, `"1."`, `"1 "`).
현재는 `strip().rstrip(".")`으로 정규화하고 있으나, 예상 외 형식이 존재할 경우
접두사 중복 출력이 발생할 수 있다.

**영향**: 대부분의 법령에서는 정상 동작하나, 비정형 응답에서 `호번호`와 `호내용`
접두사가 이중으로 출력될 가능성 있음.

**해결 방향**: 캐시된 XML 전수 조사로 `호번호` 필드의 실제 형식 분포를 확인하고,
필요 시 정규화 로직 보강.

## 3. 캐시 파일명 해시 시 역추적

법령명이 255바이트를 초과하는 경우 SHA-256 해시 접미사를 붙여 파일명을 생성한다.
이 경우 파일명만으로는 원본 법령명을 완전히 복원할 수 없다.

**영향**: `.cache/history/` 디렉토리에서 파일을 수동으로 찾을 때 불편.

**해결 방향**: 필요 시 `.cache/history/_index.json` 매핑 파일 도입 검토.
현재는 코드 내에서 법령명 → 해시가 결정론적이므로 운영상 문제 없음.

## 4. 부칙 내 들여쓰기 텍스트의 코드블럭 변환

부칙(`addenda`) 본문은 API 원본 텍스트를 그대로 출력한다. 원본에 4칸 이상
들여쓰기가 포함된 경우, 빈 줄 뒤에 오면 CommonMark에서 인덴트 코드블럭으로
해석된다.

**영향**: 민법 부칙 등에서 하위 항목 텍스트가 코드블럭(`<pre><code>`)으로
렌더링되어 의도와 다른 표시가 됨.

**해결 방향**: 부칙 본문 출력 시 선행 공백을 strip하거나, 들여쓰기를
Markdown 리스트 문법으로 변환하는 후처리 추가.

## 5. `<img>` 태그 처리

API 응답 XML에 `<img>` 태그가 포함된 법령이 410개 파일, 총 1,582건 존재한다.
두 가지 유형이 있다:

- **ID only** (779건): `<img id="9818853"></img>` — src 속성 없이 ID만 존재.
  law.go.kr의 JS 없이는 렌더링 불가. 현재 깨진 이미지 또는 빈 요소로 표시됨.
- **src 포함** (803건): `<img src="http://www.law.go.kr/flDownload.do?flSeq=...">` —
  law.go.kr 내부 URL. 외부에서 접근 가능하나 안정성 보장 불가.

**영향**: 수식, 별표, 서식 등을 이미지로 표현하는 법령(시행령, 시행규칙 등)에서
내용 누락이 발생함.

**해결 방향**:
1. ID only → `flDownload.do?flSeq={id}` URL로 변환하여 src 통일
2. 이미지를 로컬 다운로드하여 저장소에 포함 (`kr/{법령명}/assets/`)하거나,
   Markdown 이미지 문법 (`![alt](url)`)으로 변환
3. 이미지 다운로드가 불가한 경우 `[별표/서식 이미지]({url})` 형태의
   플레이스홀더로 대체

## 6. 꺾쇠 괄호 텍스트의 HTML 해석

부칙의 `부칙 <제1395호,1995.10.27>`, `<20>부터 <138>까지 생략`,
본문의 `<개정 2024.9.20>`, `<신설 1984.4.10>` 등 꺾쇠 괄호(`< >`)가
HTML 태그로 해석되어 렌더링 시 사라질 수 있다.

**영향**: 개정/신설 주석, 부칙 번호 등이 표시되지 않음.

**해결 방향**: 꺾쇠 괄호를 HTML 엔티티(`&lt;`, `&gt;`)로 이스케이프하거나,
법령 특유의 패턴을 인식하여 괄호 등 다른 기호로 치환.

## 7. API XML 파싱 실패 (mismatched tag)

일부 법령의 특정 개정본에서 국가법령정보센터 API가 잘못된(태그가 불일치하는)
XML을 반환하여 파싱에 실패한다. 해당 MST는 캐시되지 않으며 rebuild 시 스킵된다.

현재 확인된 6건:

| MST | 법령명 | 제개정구분 | 공포일자 | 공포번호 |
|-----|--------|-----------|----------|----------|
| 178528 | 선거관리위원회 공무원 규칙 | 일부개정 | 2015-12-29 | 00437 |
| 184279 | 선거관리위원회 공무원 규칙 | 일부개정 | 2016-06-24 | 00452 |
| 192860 | 선거관리위원회 공무원 규칙 | 일부개정 | 2017-03-24 | 00461 |
| 209096 | 선거관리위원회 공무원 규칙 | 일부개정 | 2019-06-25 | 00498 |
| 229501 | 어선원 및 어선 재해보상보험법 시행령 | 타법개정 | 2021-02-17 | 31453 |
| 233723 | 국토의 계획 및 이용에 관한 법률 시행령 | 일부개정 | 2021-07-06 | 31877 |

**영향**: 해당 법령의 위 개정본이 Git 이력에서 누락됨. 같은 법령의 다른
개정본(현행본 포함)은 정상.

**해결 방향**: API 원본 XML을 수동 확인하여, 태그 불일치를 보정하는 전처리
(예: `lxml`의 `recover=True` 모드) 도입을 검토.
